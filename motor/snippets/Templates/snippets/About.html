About the website

This final degree project by the Polytechnic University of Catalonia had as a primary objective the development of a web application that would allow its users to see in real time the data gathered by the sensors of the Stirling Dish Generator installed at the rooftop of the ETSEIB building, in Barcelona. See location.

The data is stored permanently and not only can it be searched both in table format and in graph format but it can also be downloaded as a "csv" file from the "Table" page to be used for analysis of the system's performance.

The site is fully responsive so it can be used from mobile devices as well.
How it works
Table
The "Table" page loads the table for today by default, changing the date on the date selector will automatically update the table.
The data to Download with the "Download CSV" button is synchronized with the table automatically so you can directly download filtered data.
The table allows filtering and reordering in each column.

The lack of a "T_Power" column is explained by the fact that the Thermal Power is a calculated value, in other words, if you were to download and use this data, you can calculate the value of T_Power with this simple product:
T_Power = Cool_Flow * ( Cool_T_out - Cool_T_in ) * 4184 / 60
Graph
The "Graph" page loads the graph for today by default, changing the date on the date selector will also automatically update the graph.
The graph selector on the left allows multiple variables to be checked at once, and the graph is synchronized with it automatically.

If more than 90 days are selected, the graph will show data points every two minutes instead of every 1 minute.
In particular, if all 13 graphs were selected for a graph of at least 7 days the data would also show only a data point every 2 minutes.
It's worth noting that every 90 graphs * day, the number of data points will be progressively reduced to optimize data transfers and computation times.
This way if we visualize 21 days and 13 graphs, we will be requesting a total of 273 graphs so since we will be over 90 graphs day we will get one datapoint every 4 minutes like so:
21 days * 13 graphs = 273 days * graphs
273 / 90 = 3.033
The "divider" is always rounded up so instead of every 3 minutes we get the points every 4 minutes.

The legend of the graph appears on the bottom, when two or more graphs are selected the arrows allow navigating the legend to see all the variables.
